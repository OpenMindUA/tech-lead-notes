# Tailor to suit the project

**TL;DR:** PRINCE2 (and almost every PM system) cannot be used
out-of-the-box. Tailor up-front *and* keep adapting throughout.

## Core ideas

- **Two flavors of tailoring required:**
    - **Initial tailoring** for systems whose default is heavy
      (PRINCE2 — maximalist).
    - **Continuous adaptation** even for systems that don't need
      initial tailoring (P3.express — minimalist).
- Tailoring is system design, not picking-and-choosing artifacts.
- Lack of tailoring is the source of the "PRINCE2 is too expensive"
  complaint.

## When this applies

- Adopting PRINCE2 (or any maximalist method) in a new context
- Periodic system retrospectives and adjustments
- Pushback from teams that the system feels too heavy or too light

## Anti-patterns

- Using PRINCE2 by-the-book for a tiny project
- Tailoring once at adoption and never revisiting
- Cherry-picking artifacts from PRINCE2 + Scrum + PMBOK without
  internal consistency
