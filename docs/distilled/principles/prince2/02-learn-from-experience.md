# Learn from experience

**TL;DR:** Reusing existing PM systems and prior-project lessons is
cheaper than reinventing them. Both **between** and **within** projects.

## Core ideas

- **Between projects:** adopt established systems (PRINCE2,
  P3.express, PMBOK Guide) instead of building from scratch.
- **Within an organization:** document lessons learned, especially
  from similar projects; make them findable and consultable.
- Requires a working mechanism for capture and sharing — without it,
  the lessons stay tacit and walk out the door with people.

## When this applies

- Project initiation: search prior similar projects
- Closing: capture lessons in a usable, discoverable form
- Designing a PMO knowledge base / lessons repository

## Anti-patterns

- "Every project is unique, lessons don't transfer"
- Lessons captured in ad-hoc docs nobody reads
- Building a methodology from zero ignoring established frameworks
