# Defined roles and responsibilities

**TL;DR:** Beyond very small projects, people need explicit clarity
on what's expected of them — and what they can expect from others.

## Core ideas

- Required as soon as the project is past trivial size.
- Reciprocal expectations: define both what each person *does* and
  what they can *demand* from others.
- A baseline for accountability, escalation, and conflict resolution.

## When this applies

- Project kick-off / charter
- Onboarding new members mid-project
- Recurring confusion or hand-off failures (the diagnostic)

## Anti-patterns

- "We're agile, we don't need roles" applied to a 30-person delivery
- Roles documented but never communicated
- Authority defined; expectations on the role-holder undefined
