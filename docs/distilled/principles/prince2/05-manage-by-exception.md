# Manage by exception

**TL;DR:** Delegate decisions within tolerances; escalate only when
thresholds are breached. Saves senior energy and creates buy-in.

## Core ideas

- **Tolerances per level.** Each management level has explicit
  thresholds (time, cost, quality, scope, risk, benefit).
- **Within → autonomous.** Below threshold the level decides without
  interference.
- **Above → escalate.** Excursions are *exceptions* that go up.
- **Why it works.** Faster decisions, more decision power for those
  closer to the work, buy-in from the delegates, scarce attention
  reserved for genuine deviations.

## When this applies

- Defining the project board / steering committee charter
- Setting up multi-team or multi-tier delivery
- Reducing PM bottlenecks on routine choices
- Drafting contracts with delegated authority

## Anti-patterns

- No tolerances → every decision escalates
- Tolerances exist but are not communicated
- Steering committee involved in routine matters
- Escalation only allowed retroactively (after a breach)
