# Manage by stages

**TL;DR:** You can only plan in detail for a short horizon. Use a
high-level plan up-front and a detailed plan only for the next stage.

## Core ideas

- **Limited foresight.** Detailed planning past a short window is
  speculation that creates false precision.
- **Rolling wave.** Outline plan for the whole project; detailed
  plan only for the *next* stage; refresh before each stage.
- Stages naturally align with **business case re-checks** (gates).

## When this applies

- Long-running or uncertain projects
- Programs with multi-year horizons
- Any project where early estimates would be heavily speculative

## Anti-patterns

- Trying to lock detailed plan to the project's last day at kickoff
- "Big upfront plan" that nobody updates
- No high-level plan, only stage-by-stage tunnel vision
