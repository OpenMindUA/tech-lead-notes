# Focus on products

**TL;DR:** Plan around the products (deliverables), not around
activities. Activity-first planning prematurely commits to one solution.

## Core ideas

- **Activity bias.** Listing activities first locks in a specific
  way of producing the result.
- **Product-first.** Identify what must be produced, evaluate
  alternative routes to produce it, *then* select the best activity
  set.
- Aligns with the PMBOK 7 distinction between **output** and
  **outcome** (product is the output that must enable the outcome).

## When this applies

- Building a Work Breakdown Structure (WBS)
- Designing a delivery plan from scratch
- Reviewing a plan that feels "activity-heavy" with vague outputs

## Anti-patterns

- WBS built as a list of tasks with no explicit deliverables
- "We always do these activities" copied from prior projects
- Optimizing the schedule before the deliverable is defined
