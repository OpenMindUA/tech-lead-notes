# Remember that a chain is only as strong as its weakest link

**TL;DR:** All PM domains interact. Over-investing in one
(typically scheduling) while neglecting others ruins the lot.

## Core ideas

- **Holistic balance.** Time, cost, scope, quality, risk,
  stakeholders, communication — all need adequate attention.
- **Common failure mode.** PMs often over-invest in scheduling while
  under-investing in other domains, which ironically makes the
  schedule itself useless.
- **Diagnostic question:** "Which area of this project is currently
  receiving the least attention?" That is the weak link to address.

## When this applies

- Reviewing where your effort is going as PM
- Designing the PM system: ensure every domain has at least basic coverage
- Audit / health check of an in-flight project

## Anti-patterns

- Lavish Gantt charts; weak risk management
- Detailed scope; nonexistent stakeholder communication
- Strong technical quality; no business case re-checking
