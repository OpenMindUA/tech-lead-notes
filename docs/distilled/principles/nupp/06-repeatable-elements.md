# Use repeatable elements

**TL;DR:** Ad-hoc work is expensive and error-prone. Build checklists
and recurring cycles so cognition can go to the actual work.

## Core ideas

- **Cost of ad-hoc.** Re-deciding routine matters every time wastes
  energy and risks missing items.
- **Checklists.** For frequently-checked items, build a checklist;
  it offloads memory and surfaces improvement ideas as you use it
  (continuous improvement).
- **Repeatable cycles.** Methodology cadences (e.g. P3.express
  monthly cycle) become second nature, freeing cognition for the
  unique parts of the project.

## When this applies

- Quality checks, risk reviews, status reports
- Onboarding/offboarding routines
- Methodology rhythm design (daily/weekly/monthly cycles)

## Anti-patterns

- Hand-crafting every status update from scratch
- Ad-hoc quality checks that miss steps under pressure
- Reinventing the wheel for routine ceremonies
