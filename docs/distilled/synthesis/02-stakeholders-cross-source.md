# Stakeholders and supporters across frameworks

**TL;DR:** All frameworks agree a stakeholder is **anyone with
interest who can impact the project** (not just the customer). They
diverge on **structure**: PMBOK 7 treats stakeholders as a
performance domain with a 5-step continuous loop; Scrum 2020
treats them implicitly via the Product Owner; **SGE elevates
Stakeholder to a 4th role** with explicit subtypes and a new
**Supporter** subtype for change agents.

## Definition convergence

> "Anyone with interest in the project who can impact it: team
> members, clients, suppliers, end users, regulators, competitors."
> — Underneath the Surface

> "An entity, individual, or group interested in, affected by, or
> impacting inputs, activities, and outcomes. Direct or indirect
> interest, inside or outside the organization." — SGE

Both extend to **inanimate stakeholders** (regulators, AI, the law).

## Structural mapping

| Framework | How stakeholders appear |
|---|---|
| **UtS / PMBOK 7 §3** | Principle: identify aggressively, communicate often, tailor channels |
| **UtS / Performance domain Stakeholders** | 5-step continuous loop: Identify → Understand → Prioritize → Engage → Monitor |
| **PRINCE2** | Project Board includes Senior User + Senior Supplier representing user / supplier groups |
| **Scrum 2020** | Implicit — Product Owner represents stakeholders to the team; "convince the PO" to change Backlog |
| **SGE** | Stakeholder is a **named role** (4th); subtypes catalogued; **Supporter** subtype for change agents |
| **DORA** | Tech lead reviews metrics with stakeholders quarterly; org governance via DORA reports |

## Subtype taxonomy (richest in SGE)

- **Customer** — receives value via purchase or selection (buyer /
  decision-maker / end-user; B2B2C / B2B2B applies)
- **User** — directly interacts with the Product
- **Decision-maker** ("chooser", Patton) — authorizes adoption
- **Legislator / policy maker** — sets rules
- **Financial sponsor** — funds; assesses viability
- **Subject matter expert** — domain knowledge; doesn't dictate
- **Governance oversight** — structures, standards, regulations
- **Supporter** (SGE) — change agent inside the organization

PMBOK 7 doesn't enumerate but matches when stakeholders are
analyzed under "Understand and analyze".

## Engagement patterns (across frameworks)

| Channel/Practice | Source |
|---|---|
| Continuous identification (don't stop at kickoff) | UtS / Performance domain |
| Tailored channel per group (formal/informal, push/pull) | UtS / Performance domain |
| Documented stakeholder register | UtS / PRINCE2 |
| Single-hub Product Owner | Scrum / SGE |
| **Focused communication** at process milestones | P3.express |
| **Monthly stakeholder satisfaction evaluation** | P3.express |
| Sprint Review as 2-way working session | Scrum 2020 / SGE |
| Adversaries-as-future-champions (engage with curiosity) | UtS PMBOK 7 §3 |
| Manage by exception — delegate within tolerances | PRINCE2 |
| Coherent governance — single point of contact per area | SGE |

## Convergent cautions

- **Stakeholder ≠ customer.** All frameworks agree.
- **Bribery is non-negotiable** even where culturally common (UtS).
- **Concealment is risky and unethical** (UtS performance domain).
- **Negative stakeholder ≠ enemy** — engage with curiosity (UtS,
  PMBOK 7 §3).
- **Justification rule:** engagement effort < gain from support
  (override only for ethics).
- **Don't bypass the Product Owner** to lobby Developers about scope
  (Scrum, SGE).

## When this synthesis applies

- Designing a stakeholder management plan that crosses methodologies
- Onboarding into an SGE-aware org coming from Scrum 2020
- Mapping subtypes from the SGE catalogue to a PMBOK-style register
- Resolving disputes about who counts as a stakeholder

## Anti-patterns (universal)

- "Stakeholder = customer" framing
- One channel for everyone
- Stakeholder register set up at kickoff, never updated
- Negative stakeholders dismissed as "always negative"
- Scrum team that forgets stakeholders are also team members and
  Supporters
