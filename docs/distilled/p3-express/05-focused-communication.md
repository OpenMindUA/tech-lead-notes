# Focused communication

**TL;DR:** P3.express ends almost every cycle with a **focused communication** — a defined message to a defined audience at a defined moment. The pattern is so universal in the manual that it appears as A10, B05, C04, E03, **and** G03. It exists to keep stakeholders aligned **without** having to chase them.

## The pattern

| When | Activity | What it announces |
|---|---|---|
| End of Project Initiation | **A10** | Project is starting; goals + benefits |
| End of Monthly Initiation | **B05** | This month's plan; updates from B01-B04 |
| End of Weekly Management | **C04** | Performance + deviations + new risks |
| End of Monthly Closure | **E03** | Satisfaction results + improvement plans |
| End of Post-Project | **G03** | Benefits realized (or not); ideas surfaced |

## How

- **Defined audience** — relevant stakeholders for that cycle (often everyone involved + interested external parties).
- **Defined channel** — banner, email, intranet announcement, virtual-team equivalent.
- **Defined content** — short, audience-tailored, **engaged in tone** (NOT dry corporate-speak).
- **Defined moment** — at the end of the cycle, immediately after the closing activity.

## Why this is its own pattern

In many organizations, projects "start and end with no clear indication" — most employees (and even managers) don't know what's happening. The result:

- Specialists focus only on their tasks
- No alignment with the project's whole goal
- Poor cross-collaboration

Focused communication fixes this with **minimum viable broadcast** at every cycle boundary. The PM doesn't have to remember to communicate — it's a scheduled activity at known points.

## Tone matters

> "You can't be at your best unless you're excited about the project, and when you are, you should be able to communicate that excitement in your communications and create a similar feeling in others."

Dry status emails defeat the purpose.

## When this applies

- Designing a project communications plan
- Coaching PMs who default to silence between events
- Deciding what + who + when for each broadcast
- Diagnosing organizational confusion ("nobody knew this project was happening")

## Anti-patterns

- **No communication between cycle boundaries** — surprises by design
- **Same generic email** every time — audience tunes out
- **Dry status report** instead of engaged narrative
- Communication **only to managers** — specialists never get aligned
- Skipping G03 because "the project is over" — closes off post-project narrative
