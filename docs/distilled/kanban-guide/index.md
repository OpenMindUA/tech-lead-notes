# The Kanban Guide — distilled

Cards distilled from [The Kanban Guide v2025.5](https://kanbanguides.org/the-kanban-guide/) — the canonical, minimal definition of Kanban as a flow-optimization strategy. Kanban is **not** a methodology or a "Sprint replacement"; it is a strategy applied to whatever workflow already exists.

## Cards

- [Three practices + Definition of Workflow](01-three-practices-and-dow.md) — define/visualize, actively manage, improve; the 6 minimum DoW elements
- [Flow metrics](02-flow-metrics.md) — the 4 mandatory metrics: WIP, Throughput, Work Item Age, Cycle Time
- [When to use Kanban (and when not)](03-when-to-use.md) — fit, complement to Sprint cadence, hand-off to DORA delivery metrics

## Cross-source connections

- **Scrum** — Kanban can be applied **inside** a Sprint; the Scrum Guide does not forbid WIP limits or flow boards. SGE adds explicit metrics commitments.
- **DORA** — Kanban's *Cycle Time* measures the same flow phenomenon as DORA's *Lead Time for Changes*, but bounded by the team's `started`/`finished` points rather than commit-to-deploy. Both rely on the same ladder discipline.
- **PMBOK 7 / UtS** — Kanban embodies the *Optimize flow* and *Continuously improve* stances called out in the Performance Domain on Project Work and Measurement.
- **Open Guide to Kanban** — the community-expanded sibling for hybrid contexts; see [`distilled/open-guide-to-kanban/`](../open-guide-to-kanban/index.md).
