# Agent index — Tier 1

A compact, one-line-per-concept catalog. Designed to fit entirely
in an LLM agent's system prompt.

> **How to use:** scan the relevant section, follow the link to a
> Tier-2 distilled card for actionable detail, or to a Tier-3 full
> chunk for verbatim text and citations.

- **Tier 1** — *this file* — index only.
- **Tier 2** — [`distilled/`](distilled/index.md) — dense cards
  with TL;DR, core ideas, when-applies, anti-patterns, cross-refs.
- **Tier 3** — full original text, kept under the chapter
  directories `01-…/` … `05-…/`.

The text below is original to this repository (CC-BY-4.0); the
underlying material is © Nader K. Rad, also CC-BY-4.0.

## PMBOK 7 — 12 principles

1. **Stewardship** — PM is a facilitator, not a technical boss. → [card](distilled/principles/pmbok-7/01-stewardship.md)
2. **Collaborative team** — build a team, not a roster of stars. → [card](distilled/principles/pmbok-7/02-collaborative-team.md)
3. **Engage stakeholders** — identify aggressively, communicate often, tailor channels. → [card](distilled/principles/pmbok-7/03-engage-stakeholders.md)
4. **Focus on value** — value = benefits ÷ cost; trace decisions to "why". → [card](distilled/principles/pmbok-7/04-focus-on-value.md)
5. **System interactions** — reason holistically; manage cognitive biases. → [card](distilled/principles/pmbok-7/05-system-interactions.md)
6. **Leadership** — leadership is behavior, not title; empower hidden leaders. → [card](distilled/principles/pmbok-7/06-leadership.md)
7. **Tailor based on context** — system design, not cherry-picking. → [card](distilled/principles/pmbok-7/07-tailor.md)
8. **Build quality in** — prevention not inspection; PM system has its own quality. → [card](distilled/principles/pmbok-7/08-quality.md)
9. **Optimize risk responses** — layered, continuous, cost-justified. → [card](distilled/principles/pmbok-7/09-risk.md)
10. **Navigate complexity** — synthesis of principles 1–3, 5, 9; no separate technique. → [card](distilled/principles/pmbok-7/10-complexity.md)
11. **Adaptability and resiliency** — predictive vs adaptive driven by product nature. → [card](distilled/principles/pmbok-7/11-adaptability.md)
12. **Enable change** — output ≠ outcome; manage adoption. → [card](distilled/principles/pmbok-7/12-enable-change.md)

→ overview: [PMBOK 7 principles index](distilled/principles/pmbok-7/index.md)

## NUPP — 6 nearly-universal principles

1. **Results over affiliations** — don't be tribal about a method. → [card](distilled/principles/nupp/01-results-over-affiliations.md)
2. **Preserve energy and resources** — Pareto; manage decision fatigue. → [card](distilled/principles/nupp/02-preserve-energy.md)
3. **Always be proactive** — projects reward proactivity (e.g. risk mgmt). → [card](distilled/principles/nupp/03-always-be-proactive.md)
4. **Weakest link** — all PM domains interact; don't over-invest in one. → [card](distilled/principles/nupp/04-weakest-link.md)
5. **Clear purpose** — counterfactual test before adopting any practice. → [card](distilled/principles/nupp/05-clear-purpose.md)
6. **Repeatable elements** — checklists and cycles free up cognition. → [card](distilled/principles/nupp/06-repeatable-elements.md)

## Methodologies — 4

- **P3.express** — open, minimalist, general; use as-is, gradual tailoring; small / medium / large; either approach. → [card](distilled/methodologies/p3-express.md)
- **PRINCE2®** — proprietary, maximalist, general; mandatory upfront tailoring; medium / large / super-large; predictive (PRINCE2 Agile for adaptive). → [card](distilled/methodologies/prince2.md)
- **Scrum** — open, framework, IT, single team ≤10; decentralized PM; adaptive only; subset coverage. → [card](distilled/methodologies/scrum.md)
- **DSDM®** — proprietary, IT, large multi-team; centralized PM; adaptive only; PRINCE2-inspired structure; MoSCoW + timebox. → [card](distilled/methodologies/dsdm.md)

→ side-by-side: [Comparison](distilled/methodologies/comparison.md)

## Scrum Guide 2020 (canonical, CC BY-SA)

Foundations:
- **Scrum definition** — lightweight framework, complex problems, immutable rules. → [card](distilled/scrum-guide/01-scrum-definition.md)
- **Theory + 3 pillars** — empiricism + lean; Transparency, Inspection, Adaptation. → [card](distilled/scrum-guide/02-theory-and-pillars.md)
- **5 Values** — Commitment, Focus, Openness, Respect, Courage. → [card](distilled/scrum-guide/03-scrum-values.md)

Scrum Team:
- **Team structure** — single team ≤10, no sub-teams, one Product Goal. → [card](distilled/scrum-guide/04-scrum-team.md)
- **Developers** — accountable for Increment + DoD + daily plan. → [card](distilled/scrum-guide/05-developers.md)
- **Product Owner** — value + Backlog management; one person, not a committee. → [card](distilled/scrum-guide/06-product-owner.md)
- **Scrum Master** — establishes Scrum + team effectiveness; leader who serves. → [card](distilled/scrum-guide/07-scrum-master.md)

Events (Sprint as container):
- **Events overview** — five events, cadence, regularity. → [card](distilled/scrum-guide/08-events-overview.md)
- **The Sprint** — heartbeat, ≤1 month, no quality decrease. → [card](distilled/scrum-guide/09-the-sprint.md)
- **Sprint Planning** — Why / What / How; ≤8h. → [card](distilled/scrum-guide/10-sprint-planning.md)
- **Daily Scrum** — 15 min, for Developers, Sprint-Goal-focused. → [card](distilled/scrum-guide/11-daily-scrum.md)
- **Sprint Review** — inspect outcome with stakeholders; not a demo. → [card](distilled/scrum-guide/12-sprint-review.md)
- **Sprint Retrospective** — plan improvements; concludes Sprint. → [card](distilled/scrum-guide/13-sprint-retrospective.md)

Artifacts (each with a commitment):
- **Product Backlog (+ Product Goal)** — single source of work; emergent. → [card](distilled/scrum-guide/14-product-backlog.md)
- **Sprint Backlog (+ Sprint Goal)** — by/for Developers; Why+What+How. → [card](distilled/scrum-guide/15-sprint-backlog.md)
- **Increment (+ Definition of Done)** — usable, additive; release independent of Review. → [card](distilled/scrum-guide/16-increment.md)
